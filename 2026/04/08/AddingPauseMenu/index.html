AddingPauseMenu

Today we had a team sync to align our milestones and distribute upcoming tasks. My primary objective for this sprint was to implement a functional Pause Menu that doubles as a network diagnostic tool for our multi-platform testing.

I successfully built the UI overlay featuring a semi-transparent gray filter and real-time network statistics, including Local IP, Host IP, and a Ping indicator. To ensure the menu doesn’t interfere with our initial connection screen, I implemented a logic gate using NetworkManager.Singleton.IsListening. This ensures the Tab key only activates the menu once a network session is actually live, keeping the starting interface clean.

The main technical hurdle was resolving the “input war” between the UI and my existing PlayerMove.cs. I handled this by utilizing a static IsPaused flag to act as a master switch. Now, when the menu is active, it completely intercepts player movement and rotation logic. This fixed a critical bug where mouse clicks were being “stolen” by the player controller, which previously prevented buttons from responding. The system is now robust, allowing for seamless transitions between gameplay and menu navigation. All changes have been pushed to the feature branch for further integration.
